## Interaction Specification

### On-Chain/Network Flows

1. **User Donation Flow**
   - Validate round ID and project ID exist
   - Check user balance sufficient
   - Estimate gas for approve + donate
   - Execute approve, then donate transaction
   - Show progress through both steps

2. **Admin Round Creation**
   - Validate dates (end > start)
   - Validate budget > 0
   - Execute `createRound()` transaction
   - Show success with round ID

3. **Round Management**
   - Activate/deactivate rounds
   - Add projects to active rounds
   - Validate project IDs unique per round

4. **Finalization (Admin)**
   - Calculate quadratic matching amounts
   - Execute `setMatchAllocations()` with results
   - Disburse funds to projects
