## Interaction Specification

### On-Chain/Network Flows

1. **Proposal Creation**
   - Select template to auto-fill form
   - Fill parameter fields
   - Submit to Snapshot (demo implementation)
   - Show success with proposal link

2. **Proposal Voting**
   - Load active proposals from Snapshot
   - Display voting options and current results
   - Cast vote through Snapshot API
   - Update vote counts in real-time

3. **Proposal Display**
   - Show proposal details, author, timeline
   - Display current voting results
   - Enable voting for active proposals

- Creation: Form â†’ Submit â†’ Success

- Template selection required
- Parameter validation
- API failures with retry
- Voting period expired
